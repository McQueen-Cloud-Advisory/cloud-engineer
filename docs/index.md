# Cloud Engineer

## Purpose

Cloud Engineer is a practical documentation site for learners who want to understand how cloud systems are planned, delivered, operated, and improved in real environments. The content is created by McQueen Cloud Advisory and is organized around foundations, provider-specific guidance, hands-on projects, reusable patterns, and career material.

## What You Should Understand

- Cloud engineering work joins architecture, security, deployment, observability, and cost control.
- Provider services matter most when they are tied to implementation decisions and operating concerns.
- A useful learning path should produce deployable systems and portfolio-ready explanations, not only service vocabulary.

## How This Fits Into Cloud Engineering

Use the site to move from core concepts to provider-specific implementation. Start with the provider-neutral foundations, choose a provider path, work through service pages as they appear in projects, and use the pattern pages to understand why an architecture is shaped the way it is.
