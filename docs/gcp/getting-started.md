# Google Cloud Getting Started

## Purpose

This page shows how to begin learning Google Cloud through the Cloud Engineer path with a focus on practical delivery work.

## What You Should Understand

- Projects, IAM, service accounts, and regions form the basic control model for Google Cloud work.
- Start with Cloud Storage, Cloud Run, Cloud Functions, Pub/Sub, Secret Manager, Cloud Monitoring, and BigQuery.
- Add Vertex AI after the core platform services feel familiar.
## Suggested First Steps

1. Create a sandbox project and review billing, IAM, and logging defaults.
2. Learn the core storage, compute, and eventing services used in the first three projects.
3. Use the analytics and AI projects to add data platform and agentic workload experience.

## How This Fits Into Cloud Engineering

Use this page to identify the first services and operating model concepts you need before moving into project work. The goal is to make the provider path manageable without treating it as a checklist of unrelated products.
