# Azure Getting Started

## Purpose

This page shows how to begin learning Azure through the Cloud Engineer path with a focus on practical delivery work.

## What You Should Understand

- Tenants, subscriptions, resource groups, and RBAC define the basic Azure operating model.
- Start with Entra ID, Blob Storage, Functions, API Management, Cosmos DB, Key Vault, Monitor, and Application Insights.
- Add analytics, container, and AI workloads after the core identity and application platform pieces are familiar.
## Suggested First Steps

1. Review tenant, subscription, identity, and cost boundaries before deploying services.
2. Use the first projects to connect functions, data services, secrets, and monitoring together.
3. Use later projects to add analytics and modern AI workload patterns.

## How This Fits Into Cloud Engineering

Use this page to identify the first services and operating model concepts you need before moving into project work. The goal is to make the provider path manageable without treating it as a checklist of unrelated products.
