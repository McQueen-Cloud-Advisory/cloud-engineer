# Amazon S3

## Purpose

Amazon S3 is AWS object storage for static assets, documents, data lake files, logs, and other durable blobs of data.

## What Problem It Solves

It provides highly durable storage for data that does not need database-style row access or attached-disk semantics.

## When to Use It

- Use it for static site assets and generated artifacts.
- Use it as a landing zone for ingestion pipelines and analytics datasets.
- Use it for backups, logs, and files that benefit from lifecycle controls.

## When Not to Use It

- Do not use it when the workload needs low-latency transactional queries.
- Do not expose buckets publicly unless the access pattern is intentionally designed for that use.

## Cloud Engineering Considerations

### Identity and Access

Control access with IAM roles, bucket policies, and narrow deployment permissions.

### Networking

Review public endpoints, VPC endpoints, and data egress paths when S3 is part of a larger application flow.

### Security

Use encryption, lifecycle rules, versioning where appropriate, and block public access unless a static hosting case requires an exception.

### Observability

Track access patterns, object growth, and storage events so data handling stays visible over time.

### Cost

Storage class choice, retention, requests, and data transfer all affect cost.
