# AWS Getting Started

## Purpose

This page shows how to begin learning AWS through the Cloud Engineer path with a focus on practical delivery work.

## What You Should Understand

- Accounts, IAM roles, regions, and cost boundaries shape how AWS environments are organized.
- Start with IAM, S3, Lambda, API Gateway, DynamoDB, Secrets Manager, CloudWatch, and EventBridge.
- Use Bedrock and analytics topics after the core delivery path feels familiar.
## Suggested First Steps

1. Set up an account or sandbox and understand identity, billing, and region choices first.
2. Practice the core serverless stack by working through storage, API, compute, and monitoring pages.
3. Use later projects to add scheduled processing, analytics, and agentic workloads.

## How This Fits Into Cloud Engineering

Use this page to identify the first services and operating model concepts you need before moving into project work. The goal is to make the provider path manageable without treating it as a checklist of unrelated products.
