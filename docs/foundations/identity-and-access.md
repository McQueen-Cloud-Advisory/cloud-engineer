# Identity and Access

## Purpose

This page explains how identities, roles, policies, and trust relationships control access to cloud resources and automation paths.

## What You Should Understand

- Human access and workload access should be modeled differently.
- Least privilege and short-lived credentials reduce blast radius.
- Identity design affects deployment pipelines, service-to-service communication, and incident response.
## How This Fits Into Cloud Engineering

Identity and access management sits underneath every cloud architecture. Weak identity boundaries can undo otherwise sound network, application, and data designs.
