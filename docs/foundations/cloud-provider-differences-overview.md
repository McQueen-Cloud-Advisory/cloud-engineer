# Cloud Provider Differences Overview

## Purpose

This page compares how major cloud providers package similar capabilities and where their operating models meaningfully differ.

## What You Should Understand

- Service names differ, but common categories such as identity, object storage, serverless compute, and monitoring exist across providers.
- Account, subscription, and project boundaries affect governance and environment design.
- Regional footprint, default integrations, and pricing models can change architecture decisions.
## How This Fits Into Cloud Engineering

A cloud engineer often has to translate a design from one provider context to another. Understanding the shared concepts and the important implementation differences makes that translation much easier.
