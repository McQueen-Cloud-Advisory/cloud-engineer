# Containers

## Purpose

This page explains why containers are used to package applications and how container platforms change deployment and runtime management.

## What You Should Understand

- Containers improve portability, dependency control, and deployment consistency.
- Images, registries, and runtime configuration are part of the application delivery path.
- Running containers in production still requires attention to security, scaling, logging, and upgrades.
## How This Fits Into Cloud Engineering

Cloud engineers use containers when they need predictable packaging or more runtime control than a function platform provides. The tradeoff is a wider operational surface area.
