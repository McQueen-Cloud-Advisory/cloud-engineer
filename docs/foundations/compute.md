# Compute

## Purpose

This page explains the main cloud compute models and the tradeoffs between virtual machines, serverless functions, containers, and managed runtimes.

## What You Should Understand

- Each compute model shifts responsibility between the platform team and the cloud provider.
- Scaling behavior, startup time, runtime constraints, and deployment model all affect fit.
- Compute choices should match workload shape, operational tolerance, and integration needs.
## How This Fits Into Cloud Engineering

Cloud engineers choose compute models based on the kind of operational burden the team can support. The right answer is usually about tradeoffs, not service popularity.
