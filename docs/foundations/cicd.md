# CI/CD

## Purpose

This page explains continuous integration and continuous delivery as the automation practices that move code and infrastructure changes safely into shared environments.

## What You Should Understand

- Build, test, validation, and deployment steps should be consistent and observable.
- Pipelines need access control, secret handling, and rollback strategies.
- CI/CD design affects delivery speed, operational confidence, and auditability.
## How This Fits Into Cloud Engineering

A cloud engineer uses CI/CD to reduce manual deployment risk and make infrastructure and application changes more predictable. It is a core part of operating cloud systems at scale.
