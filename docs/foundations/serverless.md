# Serverless

## Purpose

This page explains serverless as an operating model where the platform manages more of the infrastructure lifecycle for the team.

## What You Should Understand

- Serverless shifts focus toward code, events, permissions, and observability rather than host management.
- Cold starts, runtime limits, and event-driven design still need careful planning.
- Serverless works best when the workload matches the platform constraints.
## How This Fits Into Cloud Engineering

Cloud engineers use serverless to reduce undifferentiated infrastructure work, but they still need to design for failure handling, deployment safety, logging, and cost visibility.
