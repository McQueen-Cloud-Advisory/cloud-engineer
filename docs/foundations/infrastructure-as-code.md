# Infrastructure as Code

## Purpose

This page covers the practice of declaring infrastructure in code so environments can be reviewed, versioned, and reproduced consistently.

## What You Should Understand

- Infrastructure changes should be reviewable and repeatable, not manual and undocumented.
- IaC helps teams standardize environments and reduce configuration drift.
- Deployment safety still depends on plan review, state handling, and access control.
## How This Fits Into Cloud Engineering

Cloud engineering depends on repeatability. Infrastructure as code turns environment creation and change management into part of the software delivery process instead of a separate manual activity.
