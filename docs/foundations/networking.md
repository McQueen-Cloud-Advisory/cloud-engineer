# Networking

## Purpose

This page covers the networking concepts that control how users, workloads, and managed services reach each other in the cloud.

## What You Should Understand

- Addressing, routing, segmentation, and DNS determine how traffic moves through a system.
- Public exposure and private connectivity create different security and reliability tradeoffs.
- Ingress, egress, and east-west traffic all affect architecture, troubleshooting, and cost.
## How This Fits Into Cloud Engineering

Cloud engineers use networking knowledge to decide how services should be exposed, isolated, and connected. Most production issues eventually involve networking assumptions somewhere in the path.
