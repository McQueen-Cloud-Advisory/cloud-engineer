# Start Here

## Purpose

This page explains how to work through the site in a practical sequence so you build durable cloud engineering judgment. The goal is to connect concepts to deployable systems instead of treating cloud learning as a list of unrelated services.

## What You Should Understand

- Foundations should come before deep provider specialization.
- Projects are where service knowledge turns into implementation skill.
- Patterns help you recognize reusable design choices across providers.

## Suggested Path

1. Read the [Foundations overview](foundations/index.md) and the most relevant core concept pages.
2. Choose a provider path: [Google Cloud](gcp/index.md), [AWS](aws/index.md), or [Azure](azure/index.md).
3. Work through the provider service pages while building the project sequence in order.
4. Use the [Career section](career/index.md) to turn project work into portfolio and interview material.

## How This Fits Into Cloud Engineering

A cloud engineer does not only memorize services. They learn how identity, networking, deployment, observability, automation, and cost interact inside a working system. This site is organized to reinforce that operating model.
