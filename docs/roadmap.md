# Site Roadmap

## Purpose

This roadmap shows how to use the site to build cloud engineering skills in a sequence that balances theory, implementation, and communication. It is meant to help learners progress from broad understanding into provider-specific delivery work.

## What You Should Understand

- The best progress comes from alternating concept study and hands-on implementation.
- Provider-specific skills make more sense after you understand shared cloud patterns.
- Portfolio projects are where architecture, security, automation, and operations come together.

## Suggested Sequence

1. Foundations: build a provider-neutral model for compute, storage, networking, identity, observability, and cost.
2. Provider Path: choose Google Cloud, AWS, or Azure and work through the getting started guide and roadmap.
3. Services and Projects: study service pages as they appear in project work rather than in isolation.
4. Patterns and Career: use pattern pages to explain design choices and career pages to present your work clearly.

## How This Fits Into Cloud Engineering

Cloud engineering is cumulative. The roadmap is designed so each new page adds more context for security, automation, troubleshooting, and architecture tradeoffs, instead of leaving those concerns until the end.
